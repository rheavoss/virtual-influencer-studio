## The Problem

Every image generated by Kling AI has two artifacts that MUST be removed before LoRA training:

### 1. Kling Watermark (bottom-right corner)
- Text: "KlingAI 3.0 Omni" + small icon
- Color: White/light gray
- Position: **Fixed** — always bottom-right corner
- Body overlap: YES — overlaps feet/ankles/lower legs in full-body shots
- Dimensions (1536×2402): approximately bottom 100px, right 340px
  → mask region: `x > 1196, y > 2302` (or last 6.5% height × last 22% width)

### 2. Navier-Stokes Inpainting Artifacts
- Caused by: using Navier-Stokes inpainting to remove the watermark (DO NOT USE cv2.INPAINT_NS)
- Appearance: Brown/orange square blotches on shoulders, chest, arms, background
- Color: HSV hue 5–22, saturation 80–255, value 60–180

---

## The Fix — LAMA Inpainting (GPU required)

**Never use cv2.INPAINT_NS or cv2.INPAINT_TELEA for watermark removal.**
They leave visible square artifacts that bake into the LoRA.

**Always use LAMA** (`simple-lama-inpainting`) on GPU (T4 minimum).

### Mask Strategy (apply BOTH masks per image):

```python
def make_full_mask(img_bgr):
    h, w = img_bgr.shape[:2]
    mask = np.zeros((h, w), dtype=np.uint8)

    # 1. Kling watermark — fixed bottom-right rectangle
    wm_h = int(h * 0.065)   # bottom 6.5% of height
    wm_w = int(w * 0.22)    # right 22% of width
    mask[h - wm_h:, w - wm_w:] = 255

    # 2. Navier-Stokes orange/brown artifacts — HSV detection
    hsv = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2HSV)
    lower = np.array([5,  80,  60])
    upper = np.array([22, 255, 180])
    artifact_mask = cv2.inRange(hsv, lower, upper)
    kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (20, 20))
    artifact_mask = cv2.dilate(artifact_mask, kernel, iterations=2)
    contours, _ = cv2.findContours(artifact_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    for cnt in contours:
        if cv2.contourArea(cnt) > 200:
            cv2.drawContours(mask, [cnt], -1, 255, -1)

    return mask
```
